# Resume Interview

An AI-powered mock interview platform. Upload a resume and job description, get tailored interview questions generated by GPT-4o, record or type answers, and receive instant feedback.

## How it works

1. **Create a job** — paste a job description and upload your resume (PDF). The backend extracts the text and GPT-4o generates targeted interview questions.
2. **Start a session** — begin a mock interview for a job. Answer each question by typing or recording audio (transcribed via Whisper).
3. **Get feedback** — GPT-4o scores each answer and returns strengths, areas for improvement, and an example answer.

## Monorepo structure

```
resume-interview/
├── frontend/   # Next.js 16 app (React 19, TypeScript, Tailwind CSS 4)
└── backend/    # FastAPI (Python), PostgreSQL, Cloudflare R2, OpenAI
```

## Local development

### Prerequisites

- Node.js 20+
- Python 3.11+
- Docker (for local Postgres + MinIO)

### 1. Start local services

From `backend/`:

```bash
docker compose up -d
```

This starts:
- PostgreSQL on port `5433`
- MinIO (S3-compatible object storage) on port `9000`, console on `9001`

### 2. Backend

```bash
cd backend
python3 -m venv .venv
.venv/bin/pip install -r requirements.txt
cp .env.example .env   # fill in values (see backend/README.md)
.venv/bin/alembic upgrade head
.venv/bin/uvicorn main:app --reload
```

API available at `http://localhost:8000`  
Interactive docs at `http://localhost:8000/docs`

### 3. Frontend

```bash
cd frontend
npm install
# create .env.local with NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY, CLERK_SECRET_KEY, NEXT_PUBLIC_API_URL
npm run dev
```

App available at `http://localhost:3000`

## Environment variables

### Backend (`backend/.env`)

| Variable               | Description                                      |
| ---------------------- | ------------------------------------------------ |
| `DATABASE_URL`         | `postgresql+asyncpg://...` connection string     |
| `CLERK_JWKS_URL`       | Clerk JWKS URL for JWT verification              |
| `OPENAI_API_KEY`       | OpenAI API key                                   |
| `R2_ACCOUNT_ID`        | Cloudflare R2 account ID                         |
| `R2_ACCESS_KEY_ID`     | R2 API token access key                          |
| `R2_SECRET_ACCESS_KEY` | R2 API token secret                              |
| `R2_BUCKET_NAME`       | R2 bucket name                                   |
| `R2_PUBLIC_URL`        | R2 public access URL                             |

### Frontend (`frontend/.env.local`)

| Variable                          | Description                        |
| --------------------------------- | ---------------------------------- |
| `NEXT_PUBLIC_CLERK_PUBLISHABLE_KEY` | Clerk publishable key             |
| `CLERK_SECRET_KEY`                | Clerk secret key                   |
| `NEXT_PUBLIC_API_URL`             | Backend API URL (e.g. `http://localhost:8000`) |

## Data model

```
Job
 ├── user_id (Clerk user)
 ├── job_title, company, job_description
 ├── resume_url (R2), resume_text
 ├── questions: [{question, resume_excerpt}]  ← GPT-4o generated
 └── Sessions[]
      └── Answers[]
           ├── question_index
           ├── answer_text, audio_url (R2)
           └── feedback: {score, strengths, improvements, example_answer}
```
